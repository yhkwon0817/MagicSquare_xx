## 세션 3 워크북 (채움 결과)

### 1) 주제 (1문장)

> **4×4 부분 마방진에서 빈칸을 채운 직후, 행·열·대각선 10선이 각각 34인지 빠르게 확정하고, 틀리면 어느 줄인지 바로 짚을 수 있게 검증 규칙과 테스트 루프를 만든다.** 미완성 격자(0 포함)는 `incomplete`로 구분한다.

*(솔루션 최소화 — 앱·솔버·UI가 아니라 **판정·확인 비용**만 다룸)*

### 2) R-G-I-O

| | 내용 |
|---|---|
| **R — Role** | 4×4 **부분** 마방진 학습자. 빈칸 2개(0)를 1~16으로 채운 뒤 **맞았는지 스스로 확인**해야 함 |
| **G — Goal** | 빈칸 채운 후 **10선 합 34 여부를 즉시 판정**하고, 틀리면 **어느 행·열·대각선**인지 식별 *(20분 낭비 → 처음·한 번에)* |
| **I — Input** | `grid: list[list[int]]` — 4×4 정수 격자. 셀 값 **0**(빈칸) 또는 **1~16** |
| **O — Output** | `validate_lines(grid)` → **ValidationResult**: `ok`, `status` (`pass`/`fail`/`incomplete`), `lines[]` — fail 시 틀린 줄 `{ id, sum, expected: 34 }`. **R5:** `0` 포함 시 `status=incomplete` |

### 3) 성공 기준 3개

| # | 성공 기준 | 연결 증거 |
|---|-----------|-----------|
| 1 | **10선×34 검증** 한 Command | ① 대각선 하나 빼먹음 |
| 2 | Test Loop Red→Green | ① 20분 낭비 → 테스트로 재현 |
| 3 | fail 시 **틀린 줄 ID** 반환 | ② 판정 못함 → 줄 단위. ③ 2차 전 항목 확인 → 1차 10선 확신 |

### 4) 표면 문제 — 하지 않을 것

- BCE 전체 / GridUI / InputHandler
- Solver / 빈칸 자동 채우기
- 설계표만, 테스트 없음
- 중복·범위 검증 (세션 4+)

### 5) 8계층 — 이번 세션

| 계층 | 할 일 |
|------|-------|
| **Rule** | R1~R5: 10선 합 = 34, 0 → incomplete |
| **Command** | `validate_lines(grid)` |
| **(Skill)** | pytest / fixture *(선택)* |
| **Test Loop** | Red(fail/incomplete) + Green(pass) |
