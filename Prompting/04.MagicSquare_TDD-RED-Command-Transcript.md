## 커맨드 핵심 내용 (tdd-red.md 발췌)

### Phase 선언

- 응답 **첫 줄**: `[RED]`
- GREEN·REFACTOR 작업 금지

### AAA 절차

1. **Arrange** — 4×4 `grid`, 시나리오 주석
2. **Act** — `result = validate_lines(grid)`
3. **Assert** — `status`, `failed_lines` (fail 시 `id`·`sum`·`expected`)

### pytest 예시 (3 시나리오)

```python
# pass — 정답 격자, failed_lines == []
# incomplete — 0 포함, status == "incomplete"
# fail — row:0 sum 33, failed_lines에 id/sum/expected
```

```bash
pytest tests/test_validate_lines.py -v
```

### 보고 형식

```markdown
[RED]

## 추가·변경 테스트
- `test_...`: (시나리오)

## pytest 결과
- 명령: `pytest tests/test_validate_lines.py -v`
- 결과: N failed, M passed

## 다음 단계
- GREEN: `src/validate_lines.py` 최소 구현
```

### 금지

- `src/` 수정
- assert 완화·삭제·skip·xfail
- 더미 구현, Solver/UI/중복·범위 검증
