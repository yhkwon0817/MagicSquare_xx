## 3. `.cursorrules` (ECB 초안)

### 3.1 Entity (E) — 도메인

- 4×4 격자, 셀 **0**(빈칸) 또는 **1~16**, 마법상수 **34**
- **10선**: R1~R4 · C1~C4 · D1(주대각) · D2(반대각)
- 줄 ID: `row:0`~`row:3`, `col:0`~`col:3`, `diag:main`, `diag:anti`
- **R5**: `0` 포함 → `status=incomplete`, 합 검증 생략
- 범위 밖: Solver, UI, BCE 전체, 중복·범위 검증

### 3.2 Control (C) — Command

```text
validate_lines(grid) -> {
  status: "pass" | "fail" | "incomplete",
  failed_lines: [{ id, sum, expected: 34 }, ...]
}
```

- pass / incomplete → `failed_lines` 빈 리스트
- 구현: `src/validate_lines.py` · 테스트: `tests/test_validate_lines.py`

### 3.3 Boundary (B) · TDD · AI

| 구분 | 규칙 |
|------|------|
| Boundary | 입출력 dict 계약, pytest 경로 준수 |
| TDD | RED→GREEN→REFACTOR; RED은 `tests/`만; skip·xfail·assert 완화 금지 |
| AI | 한국어; TDD 시 첫 줄 `[RED]`/`[GREEN]`/`[REFACTOR]`; commit은 사용자 요청 시만 |

`Report/03`의 `ValidationResult`(ok + lines[])와 `.cursorrules`의 `failed_lines` 명칭은 동일 계약 — fail 시 틀린 줄만 반환.
