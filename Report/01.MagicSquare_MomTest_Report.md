## 3. 표면 문제 (금지 — 솔루션 섞인 문장)

인터뷰·과제 맥락에서 드러난 **잘못된 문제 정의** (산출물·도구 중심):

> *「4×4 격자에 빈칸 2개를 채워 행·열·대각 합 34인 **마방진을 만드는 프로그램**을 작성한다.»*  
> *「ECB로 **Entity/Control/Boundary**를 나눠 **GridUI·Solver**를 구현한다.»*

**누락되는 것**:

- 1차 시도에서 **검증이 불완전**할 때의 **시간·재작업 비용**
- “맞다”고 **확신할 수 있는 신호** (2차엔 전 항목 수기 확인에 의존)
- 부분 검증 통과와 **전체 마방진** 통과의 구분

## 4. 진짜 문제 (한 문장)

> **부분 마방진을 처음 맞출 때, 행·열·대각을 빠짐없이 검증했다는 확신이 없어 시간을 쓰고 표를 다시 그린 뒤에야, 손으로 전 항목을 다시 더해 “맞다”고 끝낼 수 있다.**

### 4.1 진짜 문제를 뒷받침하는 관찰

| 관찰 | 근거 ID |
|------|---------|
| **대각 누락**이 **~20분** 비용으로 이어짐 | F-2 |
| 오류 발견은 **혼자 재검산**에 의존 — 외부 피드백 없음 | F-4 |
| 수정 전략이 **표 전면 재작업** | F-5 |
| “맞다”는 **2차에만** — 행·열·대각·중복·범위 **전부 수기 확인** 후 | F-6 |
| 최종 **제출·정답**은 달성했으나, **1차 시도만으로는 불가** | F-6, F-7 |

### 4.2 Mom Test 증거 3줄

1. **대각선 하나를 빼먹어 약 20분**을 날렸다. (F-2)  
2. **혼자 다시 더해** 알고 **표를 처음부터 다시** 그렸다. (F-4, F-5)  
3. 두 번째엔 **행·열·대각·중복·범위까지** 확인한 뒤 **제출·정답**으로 끝났다. (F-6, F-7)
